Agroman — Spring Boot 4 + React/Vite skeleton

Stack
- Backend: Spring Boot 4.0.x, Java 25, Gradle (Kotlin DSL), REST, Spring Data JPA, PostgreSQL, Flyway
- Tests: JUnit Jupiter 6.x, Spring Boot Test, Testcontainers (PostgreSQL)
- Frontend: React + Vite + TypeScript, runtime config via env.js, served by Nginx
- Database: PostgreSQL (docker-compose service provided)

Repository layout
- backend/ — Spring Boot app (standalone Gradle project)
- frontend/ — Vite React app
- docker-compose.yml — Minimal stack (PostgreSQL)

Backend (dev)
1. cd backend
2. ./gradlew bootRun --args='--spring.profiles.active=dev'
- Runs at http://localhost:8080
- Base path /api (health at /api/health)
- Dev DB expects localhost:5432, db/user/pass agroman

Tests
1. cd backend
2. ./gradlew test

Frontend (dev)
1. cd frontend
2. npm install
3. npm run dev
- Vite at http://localhost:5173
- Reads backend API base from public/env.js (defaults to http://localhost:8080/api)

Docker images
- Backend: cd backend; docker build -t agroman-backend:dev .
- Frontend: cd frontend; docker build -t agroman-frontend:dev .
  - Override API_BASE_URL at runtime with -e API_BASE_URL=...

Docker Compose (minimal DB only)
1. docker compose up -d
- PostgreSQL on localhost:5432 with db/user/pass agroman

Notes
- Dev CORS allows http://localhost:5173
- Flyway baseline: backend/src/main/resources/db/migration/V1__baseline.sql
- Frontend runtime env generated by frontend/ops/nginx/entrypoint.sh inside container
